Dashboard de Análisis de Activos Financieros

API REST + dashboard web que expone una versión reducida del sistema de análisis de activos de la tesis de grado: monitoreo de CEDEARs, señales de análisis técnico real (RSI / MACD / SMA) y un slash command propio que dispara los análisis desde Claude Code.

Stack: FastAPI + SQLModel + SQLite
Autor: Erlin Rey

Proceso de desarrollo · Superpowers

De la idea al merge: un pipeline con artefactos versionados

Cada etapa deja un artefacto versionado en docs/superpowers/ (specs y plans con fecha). La integración a main es siempre vía Pull Request; nunca se commitea directo.
Spec antes que código Diseño

El brainstorming se cristaliza en una spec de diseño que fija alcance, decisiones y sus razones — p. ej. score-tecnico-cartera-on-demand-design.md. Donde la spec y la propuesta difieren, manda la spec.

Plan con TDD Ejecución

El plan baja la spec a tareas con checkboxes y tests primero (rojo → verde). Un plan real: backend-api.md, los 7 endpoints construidos endpoint por endpoint con su test.

Verify + PR Cierre

Nada se da por hecho sin evidencia: uv run pytest en verde antes de cerrar. El branch feature/… se integra a main por PR (GitHub Flow).

Configuraciones de Claude Code

Qué gobierna al agente, y en qué nivel

CLAUDE.md Context
Contexto jerárquico cargado por sesión: stack, estructura del repo y las reglas de arquitectura invariantes.
.claude/rules/ Context
Reglas inyectadas por glob: backend.md al tocar backend/**, frontend.md en frontend/**. El contexto correcto, solo cuando aplica.
settings.json Harness
allow / deny / env: límite determinístico del runtime. Habilita pytest/uvicorn, bloquea rm -rf y push --force, fija PYTHONPATH.
skill /tecnico Harness
Slash command propio que materializa el ciclo ReAct (think → act → observe) y delega el cálculo a scripts/score_tecnico.py.
El límite determinístico vive fuera del prompt settings.json
"allow": [ "Bash(uv run pytest *)",
           "Bash(uvicorn backend.main:app *)" ],
"deny":  [ "Bash(rm -rf *)",
           "Bash(git push --force*)" ],
"env":   { "CORS_ORIGINS": "http://localhost:3000",
           "PYTHONPATH": "." }

Recursos técnicos del repositorio

Los archivos que definen el proyecto

CLAUDE.mdcontexto + arquitectura
# Reglas de arquitectura
- El backend es solo CRUD. Nada de
  yfinance ni pandas en backend/;
  todo lo que descarga precios o
  calcula indicadores vive en scripts/.
- openapi.yaml es la fuente de verdad.
.claude/rules/backend.mdregla por glob
---
paths: ["backend/**/*.py"]
---
- El backend no toca el dominio
  externo: no importa yfinance ni
  pandas. El backend es solo CRUD.
.claude/rules/frontend.mdregla por glob
---
paths: ["frontend/**/*.{html,js,css}"]
---
- Vanilla single-file: todo en
  frontend/index.html. Sin frameworks,
  sin build, sin node_modules.
.claude/settings.jsonpermisos del harness
"allow": ["Bash(uv run pytest *)", …],
"deny":  ["Bash(rm -rf *)",
          "Bash(git push --force*)"],
"env":   {"CORS_ORIGINS":
            "http://localhost:3000",
          "PYTHONPATH": "."}
skills/tecnico/SKILL.mdslash command ReAct
---
name: tecnico
description: análisis técnico
  (RSI/MACD/SMA, score 0-100)
---
