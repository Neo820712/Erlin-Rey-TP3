Exposición de Propuesta (Clase 5)

Visualizador de CEDEARs con Análisis Técnico

Sistema full-stack asistido por IA para el monitoreo de CEDEARs en el mercado local, la ingesta automática de datos históricos desde Yahoo Finance y la graficación de indicadores técnicos calculados al vuelo en el cliente.

Materia: Desarrollo de Software Asistido por IA (CEIA)
Tecnología Base: FastAPI + SQLite + Chart.js
Expositor: Erlin Rey
Expositor
El Concepto y Alcance

Visualizador CEDEARs: Simple, Enfocado y Real

Foco en CEDEARs de Wall Street Activos

La aplicación realiza un seguimiento específico de CEDEARs populares (ej: AAPL, TSLA, MSFT, AMZN). El usuario interactúa dando de alta activos por su ticker en la lista de monitoreo.

Ingesta de Datos Reales Integración

Se descarta la simulación de precios. El sistema utiliza una API pública y gratuita de Yahoo Finance para descargar precios de cierre históricos reales, demostrando una integración real.

Arquitectura del Flujo de Datos
  • Frontend Single-File: Envía el ticker ingresado en el formulario al backend local.
  • Persistencia SQLite: El backend valida el activo y crea su estructura relacional.
  • Ingesta vía Skill: El slash command descarga los precios históricos de Yahoo Finance y los almacena.
  • Cálculo en Cliente: Al cargar, el frontend procesa la serie de precios para proyectar los gráficos e indicadores.
Visualización e Indicadores

Gráficos Interactivos y 4 Features Técnicas

Visualización con Chart.js Gráfico

Gráfico dinámico de línea temporal con el precio del CEDEAR e indicadores seleccionables. La leyenda del gráfico permite ocultar o mostrar cada línea con un clic para análisis interactivo.

Cálculo en el Cliente: Desarrollado en JavaScript puro para procesar los datos descargados instantáneamente.

📊 Estrategia de Cruces: La combinación de SMA 20, 50 y 200 provee alertas de "Cruce Dorado" y "Cruce de la Muerte" directamente en el navegador.
SMA 20 y 50 Tendencia

Medias móviles de corto y mediano plazo. Identifican el soporte dinámico del precio en tendencias activas.

SMA 200 Largo Plazo

La media móvil institucional de 200 días. Define la frontera definitiva entre mercados alcistas y bajistas.

RSI (Oscilador) Momento

Índice de Fuerza Relativa (0-100). Detecta zonas críticas de sobrecompra (>70) o sobreventa (<30).

MACD Cruce

Convergencia/Divergencia de medias exponenciales. Identifica aceleraciones de precio y puntos de giro.

Justificación de la Arquitectura

El Stack de Tecnologías Elegido

FastAPI + SQLModel

Capa Backend

FastAPI implementa un contrato estricto de API REST. Gracias a SQLModel, se combina la estructura de tipos de Pydantic con la persistencia ORM de SQLAlchemy en un único modelo, simplificando la sincronización de datos.

HTML/CSS/JS Single-File

Capa Frontend

Apegado a la metodología del curso para desarrollo asistido por IA: sin pasos de compilación (Babel/Webpack/Vite), lo que simplifica la lectura del asistente de IA. Un único objeto global state maneja el estado de la UI.

SQLite

Capa Persistencia

Base de datos relacional compacta que persiste todo en el archivo local data/activos.db. Soporta relaciones de clave foránea (1 CEDEAR a muchos registros de precio) sin requerir servicios de red dedicados.

Marco Teórico del Curso

Alineación Metodológica: Clases 2 y 3

Clase 2 — El "Triángulo de JS" en el Frontend Frontend

La arquitectura del dashboard de CEDEARs aplica rigurosamente el modelo teórico de la Clase 2:

  • Estado: Concentrado en un objeto global reactivo único (state) que almacena los tickers y datos de series temporales.
  • DOM: Sin mutaciones directas distribuidas. El refresco visual se canaliza a través de una única función central render().
  • Eventos: Los listeners interactivos mutan el estado y disparan la actualización del DOM de manera secuencial.
Clase 3 — Backend REST y Persistencia Relacional Backend & DB

Diseño robusto guiado por el marco teórico del desarrollo cliente-servidor:

  • Contract-First: El contrato de la API está definido en openapi.yaml antes de escribir una línea de código.
  • SQLite Relacional: Estructura de tablas SQL vinculadas mediante Claves Foráneas (Relación 1-a-N entre CEDEARs y Precios).
  • Observabilidad: Uso riguroso de códigos de estado HTTP (201 para creación, 204 para eliminación) y manejo de errores.
